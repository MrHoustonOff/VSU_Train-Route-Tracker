# Train Route Tracker

Система учёта и визуализации маршрутов пассажирских поездов.

## Описание проекта
Проект реализует трёхслойную архитектуру (UI -> Service -> Repository) для управления данными о железнодорожных маршрутах и их станциях.

## Технологии
- **Java 17**
- **Maven 3.9+**
- **H2 Database** (JDBC)
- **Lombok**
- **Servlets / JSP**
- **Leaflet.js** (Карты)

## Схема базы данных

Проект использует реляционную базу данных H2. Схема состоит из двух связанных таблиц:

### 1. Таблица `routes` (Маршруты)
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `id` | BIGINT (PK) | Уникальный идентификатор (Auto Increment) |
| `train_number` | VARCHAR(50) | Номер поезда (например, "062М") |
| `name` | VARCHAR(255) | Название маршрута |

### 2. Таблица `stations` (Станции)
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `id` | BIGINT (PK) | Уникальный идентификатор (Auto Increment) |
| `route_id` | BIGINT (FK) | Ссылка на маршрут (`ON DELETE CASCADE`) |
| `name` | VARCHAR(255) | Название станции |
| `arrival_time` | TIME | Время прибытия |
| `departure_time` | TIME | Время отправления |
| `order_index` | INT | Порядковый номер в пути |
| `day_offset` | INT | Смещение дня пути (0 - старт, 1 - след. день) |
| `type` | VARCHAR(20) | Тип (DEPARTURE, INTERMEDIATE, ARRIVAL) |
| `latitude` | DOUBLE | Широта (для карты) |
| `longitude` | DOUBLE | Долгота (для карты) |

## Как запустить (Гайд)

### Шаг 1: Наполнение тестовыми данными
Чтобы база данных не была пустой и вы сразу увидели маршруты на карте, запустите утилиту-сидер:
```bash
mvn compile exec:java -Dexec.mainClass="com.trains.util.DataSeeder"
```

### Шаг 2: Запуск WEB-интерфейса (Основной режим)
Приложение работает на встроенном сервере Tomcat. Для запуска выполните:
```bash
mvn tomcat7:run
```
После появления сообщения `INFO: Starting ProtocolHandler ["http-bio-8080"]`, откройте браузер:
 **[http://localhost:8080/](http://localhost:8080/)**

### Шаг 3: Запуск Консольного интерфейса (Дополнительно)
Если требуется проверить работу через терминал:
```bash
mvn compile exec:java
```

## Особенности реализации (Для защиты)
- **Чистая Java**: Не использовались тяжелые фреймворки (Spring/Hibernate).
- **Слоистая архитектура**: Логика отделена от хранения и представления.
- **Интерактивность**: Карта Leaflet.js автоматически отрисовывает путь по координатам из БД.
- **Безопасность**: Защита от SQL-инъекций через PreparedStatement.
